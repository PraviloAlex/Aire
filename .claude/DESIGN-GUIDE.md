# Aire — Design Guide (single source of truth)

> Любой прогон ECC, который трогает UI, ОБЯЗАН соблюдать этот файл. Цель — «дорого, но просто»: контейнеры, контраст, одинаковые отступы. Эффекты — вишенка, не фундамент.

## Главный принцип
Полировка = **видимые контейнеры + контраст + единые отступы + чистые контролы**. Сначала это, потом свечение/градиенты. Если карточка сливается с фоном — экран выглядит сыро, каким бы ни был цвет/шрифт.

## Поверхности и контраст (тёмная тема)
- Фон экрана: атмосферный через `ScreenBackground` (база ~`#070A11`/`#0B0F1A`).
- Карточка (обычная): фон `rgba(255,255,255,0.06)`, граница `rgba(255,255,255,0.12)`.
- Карточка (выделенная/нажатая ховер): фон `rgba(255,255,255,0.10)`, граница `rgba(255,255,255,0.16)`.
- Цветная карточка состояния: фон `rgba(255,255,255,0.06)` + граница `${accent}33` (≈20%), акцент — в орбе/иконке, НЕ в еле видимой рамке.
- ПРАВИЛО: поверхность карточки должна явно читаться светлее фона. Запрещены фоны < `0.05` и границы < `0.10` — они исчезают.

## Скругления (radius)
- Карточки: `radius.md` (20). Не делать слишком круглыми (28+) — выглядит «пузырём».
- Чипы/пилюли: `radius.pill`.
- Иконочные подложки: `radius.sm`–`radius.md` (12–16).
- Никогда не скруглять одиночные стороны с `border-left/top` — только полная рамка.

## Отступы (через токены spacing)
- Поля экрана: `spacing.xl` (24) по горизонтали; верх 26–28; низ 104 (под таб-бар).
- Паддинг внутри карточки: `spacing.lg` (16) — ОДИНАКОВО во всех карточках.
- Зазор между карточками: 12–14. Между секциями: 20–24. Элементы внутри карточки: 8–10.
- Контент НИКОГДА не обрезается; горизонтальные ряды — скролл с `paddingRight`, последний элемент не липнет к краю.

## Типографика
- Шрифт: Manrope (на вебе через Google Fonts, кириллица). Натив — системный.
- Заголовок экрана: 26–30 / 800 / `letterSpacing: -0.5` / цвет `#F6F1E9`–`#EAF0F8`.
- Подзаголовок секции (eyebrow): 11 / 700 / UPPERCASE / `letterSpacing: 1.5` / muted.
- Заголовок карточки: 16 / 700.
- Тело: 14 / 400 / lineHeight 20.
- Подпись/мелочь: 12–13 / 500 / muted.
- Только веса 400/500/700/800. БЕЗ подчёркиваний и декоративных линий под текстом.

## Цвет / акценты
- Один акцент на контекст: цвет состояния из `stateColors` (calm cyan, focus blue, fear amber, recover green-teal, sleep violet). Паника/SOS — `colors.danger` (красный).
- Текст на цветной заливке — самый тёмный оттенок того же цвета, не чёрный.

## Кнопки и контролы
- Основная кнопка: заливка акцентом, текст тёмный (`#06121F`), высота 52–56, `radius.md`, центр. Нажатие: `opacity 0.9` или `scale 0.98`. Disabled: `opacity 0.5`.
- Вторичная: прозрачный фон, граница `rgba(255,255,255,0.14)`, текст светлый.
- Сегмент-контрол/табы: пилюли; выбранная = заливка акцентом (или surface+граница акцента), невыбранные = surface + слабая граница.
- Иконки: Ionicons **outline**, размер 20–24; в активном табе можно filled-вариант.

## Орб (фирменный элемент)
- Светящийся шар: яркий центр → цвет состояния → ПРОЗРАЧНОСТЬ. БЕЗ тёмного цвета-края и без жёсткого кольца — растворяется в фоне.
- В сессии (с текстом внутри): центр приглушённый, чтобы белый текст читался; свечение мягкое, без тёмной полосы до кольца прогресса.
- Веб-свечение: `boxShadow` ТОЛЬКО на круглом элементе (`borderRadius: size/2`), иначе будет квадратная тень.

## Чего НЕ делать (частые ошибки)
- Не делать «висящие» иконку+текст без карточки-контейнера.
- Не оставлять декоративные линии/«подчёркивания» под подписями.
- Не начинать с эффектов (glow/gradient) при невидимых карточках.
- Не имитировать тени/blur в нативе (только лёгкий `boxShadow` под `Platform.OS==='web'`).
- Не обрезать текст; выравнивать отступы по сетке выше.

## Эталон
Ориентир по уровню полировки — присланный владельцем UI (KukuPaste): чётко видимые карточки, аккуратная группировка, чистые пилюли/кнопки. Сверяться визуально в превью (`npm run build && npm run preview`), а не только по typecheck.
